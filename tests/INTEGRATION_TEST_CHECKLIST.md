# Integration Test Checklist

## 整合測試清單

執行這些測試來驗證系統是否正常運作。

### 前置條件
1. Flask 後端正在運行
2. 在瀏覽器中開啟 frontend/index.html

---

## ✓ 11.1 啟動 Flask 後端伺服器

**步驟:**
```bash
cd backend
source .venv/bin/activate
python app.py
```

**預期結果:**
- 伺服器在 http://localhost:5000 啟動
- 顯示 "Loaded 30 words from words.json"
- 沒有錯誤訊息

**狀態:** ✅ PASSED

---

## ✓ 11.2 在瀏覽器中開啟 frontend/index.html

**步驟:**
```bash
# 方法 1: 使用 Python 簡易伺服器 (推薦)
cd frontend
python3 -m http.server 8000
# 然後在瀏覽器開啟 http://localhost:8000

# 方法 2: 直接開啟檔案
# 在瀏覽器中開啟 file:///path/to/frontend/index.html
```

**預期結果:**
- 頁面成功載入
- 顯示標題 "注音輸入法練習"
- 沒有 JavaScript 錯誤

**狀態:** ✅ PASSED

---

## ✓ 11.3 驗證鍵盤渲染所有符號

**測試步驟:**
1. 檢查頁面上是否顯示虛擬鍵盤
2. 確認鍵盤包含所有注音符號：
   - 聲母：ㄅㄆㄇㄈㄉㄊㄋㄌ...
   - 韻母：ㄧㄨㄩㄚㄛㄜㄝ...
   - 聲調：ˊˇˋ˙ 和空格(一聲)

**預期結果:**
- 鍵盤正確渲染為多行
- 每個按鍵顯示注音符號和對應的鍵盤字母
- 空格鍵顯示為寬按鈕，標示 "(一聲)"

**狀態:** ✅ PASSED

---

## ✓ 11.4 驗證當前字詞顯示

**測試步驟:**
1. 觀察練習區域
2. 確認顯示中文字

**預期結果:**
- 顯示一個大的中文字（如：你、好、我等）
- 字下方顯示對應的注音符號
- 沒有顯示 "載入中..." 或錯誤訊息

**狀態:** ✅ PASSED

---

## ✓ 11.5 按有效的注音鍵並驗證高亮

**測試步驟:**
1. 查看當前字詞的注音（例如：ㄋㄧˇ）
2. 按下對應的鍵盤按鍵（例如：s）
3. 觀察虛擬鍵盤的變化

**預期結果:**
- 按下的鍵在虛擬鍵盤上高亮顯示（綠色背景）
- 高亮效果在 200ms 後自動消失
- 主控台顯示 "Correct input: s"

**狀態:** ✅ PASSED

---

## ✓ 11.6 完成字詞並驗證載入下一個字

**測試步驟:**
1. 完整輸入當前字的注音序列
   - 例如：「你」需要輸入 s, u, 3
2. 觀察字詞變化

**預期結果:**
- 輸入完最後一個符號後，顯示 "Word complete!"
- 自動載入下一個不同的中文字
- 新字的注音也更新顯示

**狀態:** ✅ PASSED

---

## ✓ 11.7 測試無效按鍵（無高亮、無錯誤）

**測試步驟:**
1. 按下非注音相關的按鍵（如：ESC, Shift, Ctrl, 英文字母如 A）
2. 觀察系統反應

**預期結果:**
- 虛擬鍵盤沒有任何按鍵高亮
- 沒有顯示錯誤訊息
- 當前練習狀態不變
- 主控台無錯誤訊息

**狀態:** ✅ PASSED
